# МСФО Экстрактор

Инструмент для автоматического извлечения четырёх форм МСФО из PDF-отчётов российских банков.
Разбирает PDF с помощью **MinerU**, определяет название банка и год отчётности, выводит результаты в **Excel** и **CSV**, предоставляет веб-интерфейс на **Streamlit**.

---

## Структура проекта

```
extract_ifrs.py        — основной обработчик: MinerU → таблицы → CSV / XLSX
metadata_extractor.py  — определение названия банка и года из content_list.json
app_ifrs.py            — Streamlit веб-интерфейс
requirements.txt       — зависимости
```

---

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install mineru[pipeline] streamlit pandas openpyxl
```

> **Важно:** MinerU требует предварительной загрузки моделей.
> После установки выполните:
> ```bash
> mineru-models-download
> ```
> Модели займут ~5–10 ГБ. Подробнее: [github.com/opendatalab/MinerU](https://github.com/opendatalab/MinerU)

### 2. Запуск веб-интерфейса

```bash
streamlit run app_ifrs.py
```

Откроется браузер по адресу `http://localhost:8501`.

### 3. Запуск из командной строки

```bash
python extract_ifrs.py путь/к/отчёту.pdf
python extract_ifrs.py путь/к/отчёту.pdf --output папка_результатов
```

## Веб-интерфейс (app_ifrs.py)

### Что умеет

| Возможность | Описание |
|---|---|
| Загрузка файлов | Несколько PDF одновременно |
| Прогресс обработки | Пошаговые статусы: MinerU → Метаданные → Загрузка |
| Определение банка | Автоматически из содержимого PDF (25+ известных банков) |
| Определение года | По паттерну «31 декабря YYYY года» в тексте |
| Просмотр форм | Вкладки с таблицами данных для каждой из 4 форм |
| Сравнение банков | Сводная таблица при загрузке нескольких отчётов |
| Экспорт | XLSX (все формы на одном листе), CSV (по формам), ZIP (архив) |

### Режимы экспорта

- **Индивидуальный** — XLSX + CSV по каждому банку отдельно
- **Сводная таблица** — количество строк по всем формам всех банков
- **ZIP-архив** — всё вместе одним файлом

## Обработчик (extract_ifrs.py)

### Что извлекает

1. **Отчёт о финансовом положении** (баланс)
2. **Отчёт о прибылях и убытках** и прочем совокупном доходе
3. **Отчёт о движении денежных средств**
4. **Отчёт об изменениях в собственном капитале**

### Алгоритм обработки

```
PDF
 └─ MinerU (parse_method=txt)
     └─ content_list.json  ←── список блоков: text / table
         ├─ Определение формы каждой таблицы
         │    ├── По caption таблицы (приоритет)
         │    └── По контексту: предшествующий заголовок
         ├─ merge_table_pages()    — склейка многостраничных таблиц
         ├─ fix_split_cells()      — исправление слипшихся ячеек (артефакт MinerU)
         ├─ fix_line_breaks_in_table()  — склейка разорванных строк
         └─ write_excel() / write_csv()
```

### Защита от ложных срабатываний

- **TOC-строки** (оглавление со страничными номерами `. 6`, `.. 12`) не задают контекст формы
- **Длинные абзацы** (> 180 символов, аудиторские заключения) не задают контекст
- **Раздел примечаний** — после обнаружения заголовка «ПРИМЕЧАНИЯ К ... 2024 ГОДА» контекстный поиск прекращается; принимаются только таблицы с точным caption

### Артефакты MinerU

- **Слипшиеся ячейки**: «3 901,6 1707,5» — два числа из разных HTML-строк попали в одну ячейку.
  Исправляется `fix_split_cells()`: первое число переносится в пустую ячейку предыдущей строки.
- **Разорванные строки**: строка, начинающаяся со строчной буквы, склеивается с предыдущей, если нет конфликта данных.

## Определение метаданных (metadata_extractor.py)

Читает первые 60 текстовых блоков из `content_list.json`.

### Год отчётности

Приоритет паттернов (от точного к общему):

1. `31 декабря 2024` — наиболее надёжный
2. `за год, закончившийся 31 декабря 2024`
3. `по состоянию на 31 декабря 2024`
4. `за 2024 год`
5. `2024 года`

Выбирается год, встречающийся чаще всего.

### Название банка

1. **Список известных банков** (25+ паттернов) — наивысший приоритет, флаг `known`
2. **Юридическое название** — паттерн `АО/ПАО «Имя»` или `Банк Имя (ПАО)` — флаг `legal`
3. **Имя файла** — крайний случай — флаг `filename`

Флаг точности отображается в интерфейсе рядом с названием банка:
- `✓ Определён точно` — из списка известных банков
- `~ Из юр. названия` — из юридической формы в документе
- `⚠ Из имени файла` — точность не гарантирована

## Известные ограничения

| Ситуация | Поведение |
|---|---|
| ВТБ «Обобщённый» отчёт | Не содержит отчёта о ДДС — форма `cash_flow` не найдена (норма) |
| Сканированный PDF | Используйте параметр `ocr` в интерфейсе (работает медленнее, ~5–15 мин) |
| Таблицы в нескольких колонках на странице | MinerU может объединять соседние колонки — рекомендуется проверить |
| Название банка не определено точно | Измените имя файла на корректное или уточните паттерн в `metadata_extractor.py` |

## Добавление нового банка в список определения

Откройте `metadata_extractor.py`, найдите список `KNOWN_BANKS` и добавьте строку:

```python
(re.compile(r'название_банка|english_name', re.IGNORECASE), 'Каноническое Название'),
```

Паттерн проверяется по первым 60 текстовым блокам PDF. Чем специфичнее паттерн — тем меньше риск ложного срабатывания.
